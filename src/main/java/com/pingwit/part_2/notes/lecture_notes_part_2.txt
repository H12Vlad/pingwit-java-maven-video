  Git
1. **Введение в Git**

   - Git — это распределенная система управления версиями,
   которая позволяет отслеживать изменения в файлах и координировать работу нескольких людей над общими проектами.
   Git отличается от других систем контроля версий, таких как Subversion (SVN),
   своей скоростью, простотой и эффективностью.



2. **Инициализация и использование**

  Основные понятия:

Репозиторий (Repository): место, где хранятся все файлы проекта и история их изменений.

Коммит (Commit): фиксация изменений в репозитории. Каждый коммит имеет уникальный идентификатор.

Ветка (Branch): параллельная версия проекта.
 Ветки позволяют работать над разными функциями или исправлениями, не влияя на основную версию проекта.

Слияние (Merge): объединение изменений из одной ветки в другую.

Конфликт (Conflict): ситуация, когда изменения в одной ветке противоречат изменениям в другой ветке.
Конфликты нужно решать вручную.
